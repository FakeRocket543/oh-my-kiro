# inff.cc Editorial Style

Style rules for feature writing on inff.cc. Active when `be columnist` or `chronicle mode`.

## Voice

The Economist meets 端傳媒. Third-person omniscient. Ironic but precise. Zero filler.

## Prohibitions

Never use these words/patterns in prose:

- 「一個」「總是」「又是」「首先」「其次」「然而」「不僅如此」
- 「值得注意的是」「無疑」「眾所周知」「不言而喻」
- 不用「也」當連接詞
- 不以「而」開頭
- 無驚嘆號
- 無 bullet points in prose body (tables and appendices OK)
- 不用「筆者」「我們」（第三人稱全知，不需自稱）

## Foreign Language Handling

- **Names**: First occurrence = full name + romanization. Thereafter = surname only.
- **Institutions/Places**: First occurrence = Chinese + original in parentheses. Thereafter = Chinese only.
- **Quotes**: Original first, Chinese translation after, source attributed.
- **Source format**: 「據 [outlet] 報導（引述 [source]）」

## Structure Rules

- No markdown bold/italic/links in article body (template doesn't parse them — use HTML or avoid)
- `## ` for h2, `### ` for h3, `---` for hr
- `![alt](url)` for images (renders as `<figure>`)
- Paragraphs separated by `\n\n`
- Series articles include HTML nav block at top

## Anti-Slop Checklist

Before declaring a draft complete:

- [ ] No prohibited words/patterns
- [ ] No sentences starting with conjunctions (而、但是、然而)
- [ ] No rhetorical questions used as transitions
- [ ] No sentences over 60 characters without a comma or dash
- [ ] Every factual claim has a source citation
- [ ] No adverb clusters (非常、極度、相當 — pick one or delete all)
- [ ] Intro paragraph hooks without throat-clearing

## Tone Calibration

| Good | Bad |
|------|-----|
| 冷陳事實，讓讀者自己判斷 | 控訴、煽情、道德高地 |
| 精確數字 + 來源 | 「很多」「大量」「普遍」 |
| 結構性解釋（制度為何如此） | 歸咎個人（誰做錯了） |
| 反直覺的並置 | 順敘 + 結論 |
| 一句一意 | 長複句塞三個子句 |
